# Security policy

Do not open a public issue containing an API key, private or signed URL, customer data, unredacted request headers, or an artifact that should not be public.

Report a vulnerability privately through [GitHub private vulnerability reporting](https://github.com/BaiqingL/latchshot/security/advisories/new). Include the affected endpoint, impact, and a minimal reproduction that uses only public non-sensitive targets.

Latchshot intentionally rejects private and special-use network targets and does not support cookies, login sessions, arbitrary browser scripts, CAPTCHA solving, or proxy rotation. A report asking to bypass those boundaries is not a vulnerability report.
